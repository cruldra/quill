title: 安装

Quill 提供了多种便捷的使用形式。

## CDN

我们提供了由 [jsDelivr](https://www.jsdelivr.com/) 支持的全球分布式 CDN。
这是开始使用 Quill 最便捷的方式,无需构建步骤或包管理器。

### 完整构建

对于大多数用户来说,完整构建是开始使用 Quill 最简单的方式。
它包含核心 Quill 库,以及常用的主题、格式和模块。

要导入完整构建,你需要包含 "quill.js" 脚本和你想使用的主题样式表。

<Sandpack
  files={{
    "index.html": `
<script src="{{site.cdn}}/quill.js"></script>
<link href="{{site.cdn}}/quill.snow.css" rel="stylesheet">

<div id="editor">
  <h2>Demo Content</h2>
  <p>Preset build with <code>snow</code> theme, and some common formats.</p>
</div>

<script>
  const quill = new Quill('#editor', {
    theme: 'snow'
  });
</script>
    `
  }}
/>

<Hint>
了解更多关于如何[自定义工具栏](/docs/formats)的信息。
</Hint>

### 核心构建

要完全自定义你的 Quill 构建,你可以导入核心库并仅添加所需的格式和模块。

<Sandpack
  files={{
    "index.html": `
<link href="{{site.cdn}}/quill.core.css" rel="stylesheet">
<script src="{{site.cdn}}/quill.core.js"></script>

<div id="editor">
  <p>Core build with no theme, formatting, non-essential modules</p>
</div>

<script>
  const quill = new Quill('#editor');
</script>
    `
  }}
/>

<Hint>
了解更多关于如何[创建自己的格式](/guides/cloning-medium-with-parchment)的信息。
</Hint>

CDN 构建将 `Quill` 暴露给全局 `window` 对象。
`Quill` 提供了一个 [`import()`](/docs/api#import) 方法,用于访问 Quill 库的组件,包括其格式、模块或主题。

## npm

如果你的项目使用 [Webpack](https://webpack.js.org/) 或 [Vite](https://vitejs.dev/) 等打包工具,建议通过 [npm](https://www.npmjs.com/) 安装 Quill。

```bash
npm install quill@{{site.version}}
```

与 CDN 方式类似,你可以从 `"quill"` 导入完整构建,或从 `"quill/core"` 导入核心构建。

```js
import Quill from 'quill';
// Or if you only need the core build
// import Quill from 'quill/core';

const quill = new Quill('#editor');
```

<Hint>
如果你想使用核心构建,请避免在整个项目中直接导入 `"quill"`。
这样做会导致完整构建,因为 `"quill"` 在导入时会注册完整构建的格式和模块。
</Hint>

`Quill.import()` 也可用于 npm 构建以访问 Quill 的库。
但是,在 npm 环境中更自然的方法是直接导入格式和模块。

```js
import Quill from 'quill';
// Or if you only need the core build
// import Quill from 'quill/core';

import { Delta } from 'quill';
// Or if you only need the core build
// import { Delta } from 'quill/core';
// Or const Delta = Quill.import('delta');

import Link from 'quill/formats/link';
// Or const Link = Quill.import('formats/link');
```

### 样式

Quill 的 npm 包也包含核心和主题的样式表,就像 CDN 构建一样。
这些样式表位于 `dist` 目录中。

如果你有适当的打包工具设置,可以在 JavaScript 文件中导入它们。

```js
import "quill/dist/quill.core.css";
```

参考 [webpack-example](https://github.com/quilljs/webpack-example) 获取在 webpack 项目中使用 Quill 的示例项目。

---
title: 为什么选择 Quill
---

自互联网诞生以来,内容创作一直是其核心。`<textarea>` 为几乎所有 Web 应用程序提供了原生且必不可少的解决方案。但在某些时候,你可能需要为文本输入添加格式。这就是富文本编辑器的用武之地。虽然有很多解决方案可供选择,但 Quill 带来了一些值得考虑的现代理念。


### API 驱动设计

富文本编辑器是为帮助人们编写文本而构建的。然而令人惊讶的是,大多数富文本编辑器并不知道用户编写了什么文本。这些编辑器通过与 Web 开发人员相同的视角来看待其内容:DOM。这带来了阻抗不匹配的问题,因为 DOM 由组织在不平衡树中的节点组成,而文本则由行、单词和字符组成。

没有以字符为度量单位的 DOM API。由于这种限制,大多数富文本编辑器无法回答简单的问题,例如"此范围内有什么文本?"或"光标是否在粗体文本上?"试图在这样的原语之上构建丰富的编辑体验是非常困难和令人沮丧的。

Quill 在设计时就考虑到了编辑和字符,并在这些以文本为中心的自然单元之上构建了其 API。要查明某些内容是否为粗体,Quill 不需要遍历 DOM 查找 `<b>` 或 `<strong>` 节点或 font-weight 样式属性&mdash;只需调用 [`getFormat(5, 1)`](/docs/api/#getformat)。它的所有核心 [API](/docs/api) 调用都允许使用任意索引和长度进行访问或修改。它的[事件 API](/docs/api/#events) 也以直观的 JSON 格式报告更改。无需解析 HTML 或对比 DOM 树。


### 自定义内容和格式

在不久前,评估富文本编辑器就像比较所需格式的清单一样简单。一个好的富文本编辑器的标志仅仅是它支持多少种格式。这仍然是一个重要的衡量标准,但下限正在接近无穷大。

文本不再是为了打印而编写的。它是为了在网络上呈现而编写的&mdash;一个比纸张更丰富的画布。内容可以是实时的、交互式的,甚至是协作的。只有一些富文本编辑器甚至可以支持图像和视频等简单媒体;几乎没有编辑器可以嵌入推文或交互式图表。然而这正是网络发展的方向:更丰富、更具交互性。支持内容创作的工具需要考虑这些用例。

Quill 公开了自己的文档模型,这是对 DOM 的强大抽象,允许扩展和自定义。Quill 可以支持的格式和内容的上限是无限的。用户已经使用它来添加嵌入式幻灯片、交互式清单和 3D 模型。


### 跨平台

跨平台支持对许多 JavaScript 库来说都很重要,但对其含义的标准往往有所不同。对于 Quill 来说,标准不仅仅是它能运行或工作,而是必须以*相同的方式*运行或工作。不仅功能是跨平台的考虑因素,用户和开发者体验也是如此。如果某些内容在 OSX 上的 Chrome 中产生特定的标记,它将在 IE 上产生相同的标记。如果在 Windows 上的 Firefox 中按 Enter 键保留粗体格式状态,它也将在移动 Safari 上保留。


### 易于使用

所有这些优势都包含在一个易于使用的包中。Quill 附带了合理的默认设置,你只需几行 JavaScript 代码即可立即使用:

<SandpackWithQuillTemplate
  files={{
    'index.js': `
const quill = new Quill('#editor', {
  modules: { toolbar: true },
  theme: 'snow'
});
`
  }}
/>

如果你的应用程序不需要,你永远不必自定义 Quill&mdash;只需享受开箱即用的丰富且一致的体验。
